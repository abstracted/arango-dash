Uploading binary data to remote locations from Foxx

Problem

My Foxx should handle binary data and upload it to another service.

Solution

ArangoDB can handle binary data using the buffer class. However, these don't mix well with strings. Therefore we have to construct the mime document by hand using buffers. We will choose the buffer to be loaded from a binary file. http bin will help us to revalidate all we did along the lines of the RFC.

var request = require("org/arangodb/request")
var fs = require("fs")
var fileBuffer = fs.readBuffer("resources/Icons/arangodb.ico");
var mimeBoundary = "1234MyMimeBoundary";
var endpoint = "http://httpbin.org/post";
var bodyStart = '--' + mimeBoundary + '\r\n'+
    'Content-Disposition: form-data; name="upload"; filename="arangodb.ico"\r\n'+
    'Content-Type: application/octet-stream\r\n\r\n';
var bodyEnd = '\r\n--' + mimeBoundary + '--\r\n';

var multipartBody = Buffer.concat([
    new Buffer(bodyStart),
    fileBuffer,
    new Buffer(bodyEnd)
]);

var opts = {
    url: endpoint,
    method: "POST",
    headers: {
        "Content-Type" : "multipart/form-data; boundary=" + mimeBoundary
    },
    body: multipartBody
};
var resp = request(opts);
console.log(resp);

Since we really want to know whats going on on the wire, we'll use sniffing. You can use the nice user interface program Wireshark. We will use the simple ngrep to output the traffic on the console to us:
